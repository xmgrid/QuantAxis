3. 趋势策略 趋势

核心思想

基于 双均线系统 判断中期上升趋势，要求短期价格在中期均线上方，且中期在长期均线上方（多头排列），同时近期价格有向上动能。

前置条件

个股必须至少有 60 个交易日 的数据，否则跳过。

判断逻辑

let last = 最新交易日数据 let prev = 5 个交易日前的数据 let ma20 = 最近 20 日收盘价均值 let ma60 = 最近 60 日收盘价均值 if (last.close > ma20) && (ma20 > ma60) && (last.close > prev.close): → 纳入趋势信号集

条件拆解

字段说明
code股票代码（6位补齐）
date交易日期
open / high / low / closeOHLC 价格
volume成交量（手）
amount成交额（元）
pct_change单日涨跌幅（%）
industry行业分类（可选列）
条件含义
close > MA20价格站上 20 日均线（短期强势）
MA20 > MA60短期均线在长期均线上方（均线多头排列）
close > close[-5]当前价高于 5 日前（短期动量向上）

理论背景

这是经典的 双均线交叉系统（Golden Cross） 变体。MA20 上穿 MA60 通常被视为中期看多信号（金叉），加上价格在短期均线上方并持续走高，三重确认趋势。5 日比较排除了刚发生金叉但随即回落的假突破。

4. 多因子选股 多因子

核心思想

通过一个 综合评分公式，在收益（动量）和风险（波动）之间做权衡，选出性价比最高的股票。

评分公式

score = 50 + (ret_20 × 1.5) − (vol × 2.0) 其中: ret_20 = (close_today / close_20d_ago − 1) × 100 // 20 日收益率 (%) vol = std(pct_change) // 日涨跌幅标准差 (波动率)

参数含义

参数值含义
基准分50中性起点，高于 50 偏多，低于 50 偏空
动量权重1.5每 1% 的 20 日收益贡献 1.5 分
波动惩罚2.0每 1 个标准差惩罚 2 分

示例计算

// 某股 20 日涨 10%，波动率 2% score = 50 + 10×1.5 − 2×2.0 = 50 + 15 − 4 = 61 ← 较强 // 某股 20 日涨 5%，波动率 8%（高波动） score = 50 + 5×1.5 − 8×2.0 = 50 + 7.5 − 16 = 41.5 ← 偏弱（波动侵蚀了收益）

选股逻辑

对所有个股计算评分后，按得分 降序排列，取前 top_n 只（默认 20，用户可调 5–50）纳入多因子信号集。

理论背景

借鉴了 夏普比率 (Sharpe Ratio) 的思想——用波动率惩罚高风险的收益。不同的是，这里使用了简单的线性加权，而非除法形式。参数 1.5 和 2.0 体现了"波动厌恶"倾向——波动惩罚权重 (2.0) 大于收益奖励权重 (1.5)。

5. 行业轮动 行业轮动

核心思想

在每个行业中选出 动量最强的股票，实现"选对行业，选对龙头"的效果。这是一种自上而下的选择方式——先锚定行业，再挑龙头。

算法

for each 行业 in 所有行业: candidates = [] // 该行业内所有个股的 (code, ret_20) for each 个股 in 行业: ret_20 = (close_today / close_20d_ago − 1) × 100 candidates.append((code, ret_20)) best = max(candidates, key=ret_20) // 取收益率最高的 if best.ret_20 > 0: // 只有正收益才纳入 rotation_set.add(best.code)

关键设计

  • 行业内择优：每个行业只选 1 只表现最好的股票
  • 正收益过滤：ret_20 > 0 确保只选有正动量的行业龙头
  • 覆盖广度：只要行业有正收益即可，不限行业数量
  • 排除无效行业：行业值为 "nan" 或空字符串的跳过

理论背景

基于 行业轮动 (Sector Rotation) 理论——不同行业在经济周期不同阶段表现各异，买入每个行业中动量最强的个股可以同时享受行业 β 和个股 α。

6. 三策略精选（交集）趋势多因子轮动

核心思想

三个策略从不同维度选股，取交集意味着股票必须 同时满足 三个独立的选股逻辑——趋势向好、性价比高、行业龙头。这构成最强信号。

all_three = trend_set ∩ rotation_set ∩ factor_set

信号叠加原理

策略维度看什么
趋势技术面均线多头排列 + 短期动量
多因子量化面收益/波动综合性价比
轮动行业面行业内最强 + 正动量

三个维度独立、不重复，交集结果天然具有较高的胜率。但也因为条件严格，可能结果较少——这是"质优于量"的设计哲学。

7. 七条件叠加过滤 7 条件

核心思想

在 三策略精选 的基础上，叠加 7 个严格的短线技术条件，进一步验证信号的即时有效性。有 严格模式 和 宽松模式 两套参数。

条件详细说明

#条件严格模式宽松模式原理
① 非 ST 股 名称不含 "ST" 排除高风险警示板股票
② 10 日涨幅 ret_10d > 5% ret_10d > 0% 确保中期处于上升通道（非下跌反弹）
③ 量比 vol_today / vol_5avg > 1.5 vol_today / vol_5avg > 1.0 今日放量——有资金关注
④ 今日涨幅区间 3% ≤ pct_change ≤ 5% 0% ≤ pct_change ≤ 7% 温和上涨——不过热也不过冷
⑤ 收于 MA5 上方 close > MA5 收盘价高于 5 日均线——短线强势
⑥ 3 日资金流入 近 3 日成交额 > 前 3 日成交额 成交额加速——资金持续流入
⑦ 今日超额成交 今日成交额 > 5 日均成交额 今日成交额高于均线——当日资金活跃

前置条件

  • 至少 30 个交易日数据（len(sub) ≥ 30）
  • 必须有 amount 列（条件⑥⑦ 依赖成交额数据）
  • 5 日均量不能为 0（避免除零）

10 日涨幅（条件②）计算

ret_10d = (close[-1] / close[-11] − 1) × 100 // 注意：是第 1 天和第 11 天的比值，即 10 个交易日的跨度

成交量（条件③）计算

volume_ratio = vol_today / mean(vol[-5:]) // 今日成交量 ÷ 近 5 日成交量均值

资金流入（条件⑥⑦）计算

// 条件⑥：近 3 日 vs 前 3 日 amt_3d = sum(amount[-3:]) amt_prev3 = sum(amount[-6:-3]) // 不足6天时用 amt_3d check: amt_3d − amt_prev3 > 0 // 条件⑦：今日 vs 5 日均值 check: amt_today − mean(amount[-5:]) > 0

严格 vs 宽松 对比

🔴 严格模式

  • 10 日涨幅 > 5%（强趋势）
  • 量比 > 1.5（显著放量）
  • 涨幅 3%–5%（精确甜区）
  • 结果极少，信号纯度极高

🟢 宽松模式

  • 10 日涨幅 > 0%（不跌即可）
  • 量比 > 1.0（不缩量即可）
  • 涨幅 0%–7%（宽泛范围）
  • 结果较多，适合扫描发现

10. 完整执行流水线

用户点击「运行分析」后的执行流程

Step 1 确定候选股票池：按行业筛选（前 n_stocks 只）或 仅限持仓
Step 2 从 K 线数据中提取候选股票的子集 df_sub，释放全量内存
Step 3 调用 compute_signals() — 对候选池中每只股票：
├─ 计算趋势条件 → trend_set
├─ 计算多因子评分 → factor_set (Top N)
├─ 计算行业轮动 → rotation_set
└─ 取交集 → all_three
Step 4 展示「三策略精选」表格
Step 5 对精选结果应用 apply_seven_conditions()，展示「7 条件叠加」结果
Step 6 根据用户勾选，分别展示趋势 / 行业轮动 / 多因子的单独结果
