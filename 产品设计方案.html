🎯 一、项目定位
一款运行在手机上的纯本地量化分析工具，不连接交易、不依赖服务器

产品定义

Product Definition
  • 面向个人投资者的本地化选股分析工具
  • 所有计算和存储均在手机本地完成
  • 专注于数据分析与选股，不做交易执行
  • 数据来源可配置，种子源为 Tushare

目标用户

Target Users
  • 有 A 股投资经验的个人投资者
  • 希望用数据辅助选股决策，而非盲从消息
  • 偏好自主掌控数据，对隐私敏感
  • 不需要复杂服务器部署的技术用户

不做什么

Out of Scope
  • ❌ 不接入券商交易接口
  • ❌ 不提供投资建议或买卖推荐
  • ❌ 不搭建后端服务器
  • ❌ 不处理全市场实时行情
  • ❌ 不做社交/社区功能
🔄 二、V1→V2 关键设计修正
基于用户反馈，从「云端交易平台」彻底转为「本地分析工具」

📋 架构变更对照表

❌ 交易执行：券商 API 对接、下单、持仓同步✅ 纯分析选股，不涉及任何交易操作
❌ 自建后端服务器：Go 微服务、API 网关、K8s 部署✅ 零服务器，全部逻辑在手机本地运行
❌ 全市场实时行情：5000+ 标的实时推送✅ 用户自定义股票范围，按需拉取数据
❌ 云端存储：PostgreSQL / Redis / TimescaleDB✅ SQLite 本地数据库，数据完全在用户手中
❌ 硬编码数据源✅ 数据源可配置，种子源默认 Tushare
❌ 固定行业分类✅ 行业划分为种子，用户可自定义分类体系
❌ 用户系统：注册登录、VIP 订阅、云同步✅ 纯本地单用户，无需登录，数据自己管理
❌ 服务端策略引擎：事件驱动架构✅ 手机端本地计算引擎，分析的股票数量用户可设

🏠 数据主权归用户

  • 所有行情数据存储在手机 SQLite
  • 用户自主决定要分析哪些股票
  • 数据导入/导出/删除完全可控
  • 无任何数据回传或上报

⚡ 手机端计算能力

  • 现代手机 CPU 足以处理百只股票的量化计算
  • 分析股票数量用户可自定义（默认 50 只）
  • 批量计算在后台线程执行，不阻塞 UI
  • 结果即时呈现，无需网络等待

🔌 数据源灵活切换

  • 默认种子：Tushare Pro API
  • 数据源接口抽象，可适配其他源（AKShare 等）
  • 支持手动 CSV/Excel 导入
  • 每日定时自动拉取更新

🔌 数据源配置

Data Source Configuration
  • 种子源：Tushare Pro（用户填入自己的 Token）
  • 可替换：实现 DataSource 接口即可接入其他源
  • K 线周期可配：日线（必选）、分钟线可选（5/15/30/60 分钟），周/月线本地合成
  • 手动导入：支持 CSV / Excel 文件导入历史数据
  • 拉取控制：用户设定每日自动拉取时间、频率限制

📊 数据范围（多周期可配）

Data Scope — Multi-Period
  • 股票池：用户自选股票列表（如 50-200 只）
  • 日线数据：每只股票的历史日线（默认 5 年），必选
  • 周线/月线：可由日线本地合成，也可直接拉取
  • 分钟线：可选配置（5/15/30/60分钟），按需拉取
  • 数据周期完全由用户控制：在设置中选择需要拉取的周期类型
  • 财务数据：季报/年报关键指标（可选）
  • 指数数据：主要大盘指数历史行情

🗂️ 分类体系

Category System
  • 种子分类：基于行业划分（申万/中信行业）
  • 用户自定义：可创建个人分类标签
  • 多标签：一只股票可属于多个分类
  • 动态分组：基于规则自动归类（如 "PE<20"）

🧠 八、本地量化分析引擎设计
运行在手机端的策略计算核心

⚙️ 引擎架构

Engine Architecture
  • 策略 DSL：JSON 格式定义条件树，前端可视化编辑，后端高效解析
  • 扫描流程：加载策略 → 遍历股票池 → 读取 SQLite 行情 → 逐条件计算 → 输出匹配结果
  • 后台执行：计算放在后台线程（Dart Isolate / RN 原生线程），不阻塞 UI
  • 分批处理：每批 20 只股票，处理完一批立即更新 UI 进度

📐 技术面因子库

Technical Factors
  • 趋势类：MA 排列/交叉/距离、MACD 信号/背离
  • 震荡类：KDJ 超买超卖、RSI 强弱、WR 威廉
  • 通道类：布林带收口/扩张/突破
  • 量价类：量比、放量/缩量、量价配合度
  • 突破类：N 日新高/新低、箱体突破
  • 所有指标本地计算，不依赖外部 API

📊 基本面因子库

Fundamental Factors
  • 估值：PE/PB/PS 及历史分位数
  • 成长：营收/净利润同比增速
  • 盈利质量：ROE/ROA/毛利率/净利率
  • 财务健康：资产负债率/流动比率
  • 分红：股息率、分红连续性
  • 需拉取财报数据（季度更新）

📱 手机端性能考量

场景数据量预估耗时策略
单只股票日线指标计算5年日线 (~1200 条)< 5ms直接计算，无需优化
日线 → 周线/月线本地合成1200 条日线 → 250 周线< 2ms/只SQL 聚合或 Dart 循环重采样，无需额外 API 调用
分钟线指标计算1年60分线 (~50,000 条)~20-50ms/只仅用户配置后才拉取和计算，默认关闭
扫描 100 只股票 × 日线策略100 × 1200 条~2-5 秒多线程分批 + 进度条 UI
单策略回测 (100只 × 5年日线)100 × 1200 条~3-8 秒后台 Isolate 执行，分批计算
多策略并行扫描 (5策略 × 200只)5 × 200 × 1200 条~10-30 秒用户设置分析数量上限，默认提示 200 只以内
💡 周线/月线本地合成策略：优先从日线数据聚合生成（按周/月 OHLC），仅当用户明确需要原始交易所周线/月线时才单独拉取。此策略可减少 40-60% 的 API 调用量和存储空间。
💡 分钟线按需拉取：分钟线数据量约为日线的 40 倍（1 年 60 分线≈5 万条），默认不拉取。用户可在设置中按周期粒度（5/15/30/60 分钟）选择性开启，系统会给出存储空间预估提示。

🤔 为什么选 Flutter？

  • Canvas 渲染：K 线图需要高性能自绘，Flutter 的 Skia 引擎比 React Native 的桥接模式更适合复杂图表
  • Dart Isolate：后台计算不阻塞 UI，天然适合批量技术指标运算场景
  • sqflite 成熟：Flutter 的 SQLite 插件生态比 RN 更稳定可靠
  • 应用体积：纯本地工具不需要 RN 的 JS Bundle 加载，启动更快
  • 备选：如果团队更熟悉 React Native，也可用 RN + 原生 Canvas 模块 + react-native-sqlite-storage
