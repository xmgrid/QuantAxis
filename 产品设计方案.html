🎯 一、项目定位
一款运行在手机上的纯本地量化分析工具，不连接交易、不依赖服务器

产品定义

Product Definition
  • 面向个人投资者的本地化选股分析工具
  • 所有计算和存储均在手机本地完成
  • 专注于数据分析与选股，不做交易执行
  • 数据来源可配置，种子源为 Tushare

目标用户

Target Users
  • 有 A 股投资经验的个人投资者
  • 希望用数据辅助选股决策，而非盲从消息
  • 偏好自主掌控数据，对隐私敏感
  • 不需要复杂服务器部署的技术用户

不做什么

Out of Scope
  • ❌ 不接入券商交易接口
  • ❌ 不提供投资建议或买卖推荐
  • ❌ 不搭建后端服务器
  • ❌ 不处理全市场实时行情
  • ❌ 不做社交/社区功能
🔄 二、V1→V2 关键设计修正
基于用户反馈，从「云端交易平台」彻底转为「本地分析工具」

📋 架构变更对照表

❌ 交易执行：券商 API 对接、下单、持仓同步✅ 纯分析选股，不涉及任何交易操作
❌ 自建后端服务器：Go 微服务、API 网关、K8s 部署✅ 零服务器，全部逻辑在手机本地运行
❌ 全市场实时行情：5000+ 标的实时推送✅ 用户自定义股票范围，按需拉取数据
❌ 云端存储：PostgreSQL / Redis / TimescaleDB✅ SQLite 本地数据库，数据完全在用户手中
❌ 硬编码数据源✅ 数据源可配置，种子源默认 Tushare
❌ 固定行业分类✅ 行业划分为种子，用户可自定义分类体系
❌ 用户系统：注册登录、VIP 订阅、云同步✅ 纯本地单用户，无需登录，数据自己管理
❌ 服务端策略引擎：事件驱动架构✅ 手机端本地计算引擎，分析的股票数量用户可设

🏠 数据主权归用户

  • 所有行情数据存储在手机 SQLite
  • 用户自主决定要分析哪些股票

  • 批量计算在后台线程执行，不阻塞 UI
  • 结果即时呈现，无需网络等待

🔌 数据源灵活切换

  • 默认种子：Tushare Pro API
  • 数据源接口抽象，可适配其他源（AKShare 等）

🧠 八、本地量化分析引擎设计
运行在手机端的策略计算核心 — 支持条件因子评估 + 内置 Vibe 2.0 算法管线

⚙️ 引擎架构

Engine Architecture
  • 策略 DSL：JSON 格式定义条件树，前端可视化编辑，后端高效解析
  • 扫描流程：加载策略 → 遍历股票池 → 读取 SQLite 行情 → 逐条件计算 → 输出匹配结果
  • 内置算法管线：Vibe 2.0 三策略 + 交集 + 七条件过滤，支持严格/宽松双模式
  • 后台执行：计算放在后台线程（Dart Isolate），不阻塞 UI，分批更新进度

📐 技术面因子库

Technical Factors
  • 趋势类：MA 排列/交叉/距离、MACD 信号/背离、ADX 趋势强度
  • 震荡类：KDJ 超买超卖、RSI 强弱、WR 威廉
  • 通道类：布林带收口/扩张/突破
  • 量价类：量比、放量/缩量、量价配合度、资金流向
  • 突破类：N 日新高/新低、箱体突破
  • 复合类：N 日收益率、涨跌幅区间、收盘价 vs MA 比较
  • 所有指标本地计算，不依赖外部 API

📊 基本面因子库

Fundamental Factors
  • 估值：PE/PB/PS 及历史分位数
  • 成长：营收/净利润同比增速
  • 盈利质量：ROE/ROA/毛利率/净利率
  • 财务健康：资产负债率/流动比率
  • 分红：股息率、分红连续性
  • 需拉取财报数据（季度更新）

📐 算法一 · 趋势策略 (Trend Following)

核心思想：基于双均线系统判断中期上升趋势，三重条件确认多头排列 + 短期动量。

判断条件（AND 逻辑）

#条件公式含义
①价格站上 MA20close > MA(close, 20)短期强势，价格在 20 日均线之上
②MA20 在 MA60 之上MA(close, 20) > MA(close, 60)短期均线高于长期均线，均线多头排列
③短期动量向上close > close[5天前]当前价高于 5 日前，价格持续走高

前置条件

个股至少需要 60 个交易日数据（由 MA60 窗口决定）。

理论背景

经典双均线交叉系统（Golden Cross）变体。MA20 上穿 MA60 通常被视为中期看多信号，加上价格持续走高，三重确认有效突破，排除假突破。

🔄 算法二 · 行业轮动 (Sector Rotation)

核心思想：在每个行业中选出 20 日动量最强的龙头股，实现"选对行业 + 选对龙头"的自上而下筛选。

算法逻辑

for each 行业 in 所有行业: candidates = [] // 行业内所有个股的 (code, ret_20) for each 个股 in 行业: ret_20 = (close[-1] / close[-20] − 1) × 100 candidates.append((code, ret_20)) best = max(candidates, key=ret_20) // 取 20 日收益率最高者 if best.ret_20 > 0: // 仅正收益纳入 rotation_set.add(best.code)

关键设计

  • 每个行业最多选出 1 只动量最强的股票（best-in-industry）
  • 仅纳入 正收益 股票（ret_20 > 0），排除下跌行业
  • 行业覆盖不设上限——只要行业有正收益龙头即可入选

🧬 算法三 · 多因子评分 (Multi-Factor Scoring)

核心思想：用一个综合评分公式在收益（动量）和风险（波动）之间做权衡，选 Top N 性价比最高的股票。

评分公式

Score = 50 + (ret_20 × 1.5) − (vol × 2.0)
ret_20 = (close[-1] / close[-20] − 1) × 100  |  vol = std(pct_change)

参数说明

参数默认值含义可调范围
基准分50中性起点，高于 50 偏多0 – 100
动量权重1.5每 1% 收益贡献 1.5 分0 – 5.0
波动惩罚2.0每 1 个标准差惩罚 2 分0 – 5.0
动量窗口20 日ret_20 的回顾天数5 – 60 日
Top N20得分前 N 名纳入因子集5 – 50

计算示例

// 某股 20 日涨 10%，波动率 2% → 得分 61 (较强)
// 某股 20 日涨 5%，波动率 8% → 得分 41.5 (高波动侵蚀收益)
// 公式体现了"波动厌恶"倾向——惩罚权重 (2.0) > 奖励权重 (1.5)

⭐ 三策略精选 (Intersection)

三策略从不同维度独立选股，取 交集 意味着股票必须同时满足三个独立逻辑——趋势向好 + 行业龙头 + 性价比高。这是最强的综合信号。

精选 = 趋势集 ∩ 轮动集 ∩ 因子集
策略维度看什么
📐 趋势技术面均线多头排列 + 短期动量确认
🔄 轮动行业面行业内最强 + 正动量
🧬 因子量化面收益/波动综合性价比 Top N

三个维度独立不重复，交集天然具有较高胜率。因条件严格，结果可能较少——「质优于量」的设计哲学。

🔍 七条件叠加过滤器 (7-Condition Filter)

核心思想：在三策略精选的基础上，叠加 7 个严格的技术面条件进行二次验证，确保信号的即时有效性。支持严格/宽松双模式，用户可以逐条件调整阈值。

七条件明细（含双模式阈值）

#条件严格模式宽松模式原理
①非 ST 股 名称不含 "ST" 或 "st" 排除高风险警示板股票
②10 日涨幅 ret_10d > 5% ret_10d ≥ 0% 确保中期处于上升通道
③量比 量比 > 1.5 量比 > 1.0 今日放量——有资金关注
④今日涨幅区间 3% ≤ pct ≤ 5% 0% ≤ pct ≤ 7% 温和上涨——不过热也不过冷
⑤收盘 > MA5 close > MA(close, 5) 收盘价高于 5 日均线——短线强势
⑥3 日资金流入 近 3 日成交额 > 前 3 日成交额 成交额加速——资金持续流入
⑦今日超额成交 今日成交额 > 5 日均成交额 当日资金活跃度超平均

公式补充

// 条件②：10 日涨幅
ret_10d = (close[-1] / close[-11] − 1) × 100   // 第 1 天到第 11 天跨度 10 个交易日

// 条件③：量比
vol_ratio = vol_today / mean(volume[-5:])

// 条件⑤：收盘 vs MA5
close > mean(close[-5:])

// 条件⑥：3 日资金流入
amt_recent = sum(amount[-3:])  >  amt_prev = sum(amount[-6:-3])

// 条件⑦：今日超额成交
amt_today > mean(amount[-5:])

前置条件

  • 至少 30 个交易日数据
  • 必须有 amount（成交额）字段
  • 5 日均量不能为零
⚙️ 十、可配置参数体系设计
核心理念：所有阈值、系数、窗口期均暴露为用户可调整的参数，内置合理默认值

🎛️ 参数设计原则

Design Principles
  • 零硬编码：所有数值型阈值均来自配置文件，引擎不内置任何魔法数字
  • 合理默认值：每个参数都有基于回测经验的出厂默认值
  • 范围约束：每个参数都有最小/最大值，防止用户设置极端不合理值
  • 克隆即修改：种子策略不可直接编辑，用户克隆副本后自由调整参数
  • 即时预览：参数调整后可快速测试，显示匹配股票数量变化

💾 持久化策略

Persistence
  • 完整策略配置（含所有参数）以 JSON blob 存入 SQLite strategies 表
  • 每条策略记录包含 schema_version 字段，支持配置格式升级迁移
  • 种子策略标记 is_seed=1，用户不可直接覆盖
  • 用户自定义策略通过 source_strategy_id 追溯来源种子
  • App 升级时种子策略参数可平滑更新，用户定制不受影响

🔄 参数版本兼容

Version Compatibility
  • 向前兼容读取：新版本引擎可读取旧版本策略配置，缺失字段自动填充默认值
  • 不删字段：JSON schema 只增不减，旧字段作为 fallback 保留至少一个大版本
  • 升级提示：用户打开旧版本策略时提示"可升级至最新参数模板"，用户自主选择

🖥️ 参数编辑 UI 设计规范

滑块控件 (Slider)

适用于：数值型连续参数
  • 显示当前值 + 单位标签
  • 左右拖动调整，实时预览匹配数变化
  • 超出默认值范围时滑块颜色变为橙色提示
  • 双击恢复默认值
  • 适用：MA 周期、权重系数、涨跌幅阈值

开关控件 (Toggle)

适用于：布尔型参数
  • iOS 风格开关，绿色=启用
  • 适用：是否启用某条件、严格/宽松模式
  • 关闭某个条件时，该条件下所有子参数灰显

分段控件 (Segmented Control)

适用于：枚举型参数
  • 2-5 个选项水平排列
  • 适用：扫描模式（行业/持仓）、严格/宽松模式
  • 切换时立即刷新参数面板显示对应模式阈值

数字输入框 (Stepper)

适用于：精确整数参数
  • − / 数值 / + 三段布局
  • 长按 ± 按钮连续增减
  • 超出范围自动钳制到边界值
  • 适用：Top N、数据天数、窗口期

🔔 参数变更即时反馈

每次参数变更后，引擎在后台以 轻量采样模式（随机抽取 10% 股票池）快速重跑策略，实时更新 UI 上显示的 "预计匹配数"。采样结果标注 "预估值"，完整扫描后替换为精确值。此设计确保参数调整体验流畅，不因完整扫描延迟（2-5 秒）而卡顿。
